Технология Flexbox

В настоящий момент наиболее популярной технологией позиционирования элементов является Flexbox. У Flexbox хорошая поддержка браузерами и на сегодняшний день это стандарт.

Идея технологии состоит в том, чтобы разделить элементы на блок-контейнер и управляемые элементы - непосредственные потомки контейнера. Блок-контейнер задает общий вид и направление группировки элементов (выравнивание в колонку или ряд, порядок появления элементов и расстояние между ними), а элементы имеют возможность задавать уникальные характеристики (встать на позицию другого элемента, изменив заданный в html порядок). Для осуществления такой гибкости в технологию введены новые абстракции - направления, вдоль которых выстраиваются элементы, линии границ блоков и элементов и т.п.

Текст как элемент flex-container. Каждый дочерний элемент flex-контейнера становится flex-элементом. Текст, который напрямую содержится в flex-контейнере, оборачивается анонимным flex-элементом. Текст, состоящий из пробельных символов, ведет себя так, словно получает свойство display: none.

Axis

Элементы flex всегда располагаются вдоль главной оси. При этом главная ось может быть расположена горизонтально (row) или вертикально (column).

  • Главная ось всегда перпендикулярна вспомогательной оси.
  • Направление главной оси можно изменить.
  • Порядок расположения элементов вдоль главной оси можно изменить.
  • Это не повлияет на направление вспомогательной оси.

Пример ниже показывает сохранение направления вспомогательной оси при смене расположения элементов вдоль главной оси. Слово start всегда прижато в этом примере к началу вспомогательной оси (top или left) вне зависимости от прямого или реверсного отображения элементов вдоль главной оси.

Свойства flex-элемента

Мощь flex раскрывается в возможности управления положением отдельного элемента и даже изменение порядка расположения элемента без какого-либо вмешательства в код HTML.

order

Это возможность управления положением отдельного элемента. Элементы располагаются в порядке возрастания значения order. При равенстве значения order порядок задается последовательностью расположения элементов в HTML. По умолчанию у всех элементов order: 1.

Свойство order влияет только на визуальное отображение. В не визуальных устройствах отображения, например системах озвучивающих содержимое сайта, этот свойство не работает.

flex-grow

Определят пропорцию значения width элемента flex контейнера, которая будет рассчитана за счет удаления свободного пространства. По умолчанию это значение для всех элементов равно 0. Отрицательные значения не поддерживаются.

На рисунке ниже видно, что при значении 1 все элементы имеют равную ширину. А при равных пропорциях значения ширины элементов равны, хотя и имеют разное значение: flex-grow - 2 для средней строки и 3 для нижней. Grow - это "рост", т.е. чем выше значение flex-grow - тем больше ширина элемента.

flex-shrink

Определят пропорцию значения width элемента flex контейнера, которая будет рассчитана за счет ширины других элементов (т.е. при отсутствии свободного пространства). По умолчанию это значение для всех элементов равно 1. Отрицательные значения не поддерживаются.

На рисунке ниже видно что при значении 1 все элементы имеют равную ширину. А при равных пропорциях значения ширины элементов равны, хотя и имеют разное значение: flex-grow - 2 для средней строки и 3 для нижней. Shrink - это "уменьшение", т.е. чем выше значение flex-shrink - тем меньше ширина элемента.

flex-basis

Определяет ширину элемента до применения flex-grow и flex-shrink. Т.е. до распределения пространства flex-контейнера. Допустимы стандартные значения размеров в px, rem, % и т.п. Ожидается поддержка дополнительных значений этого свойства: content, max-content, min-content, fit-content и т.п.

flex

Краткая форма записи для перечисленных выше flex-grow, flex-shrink и flex-basis. Обязательным для объявления является только flex-grow.

align-self

Свойство, позволяющее изменить правило align-items для отдельного элемента. К сожалению доступно всего два значения - flex-start и flex-end.

CSS-тени

Тень под текстом или элементом обычно применяется для создания эффекта трёхмерности, привлечения внимания к элементу или как часть дизайна. Небольшая тень под элементами придаёт также странице объём и глубину.

text-shadow

Свойство text-shadow используется для добавления тени к тексту. Тень текста позволяет создавать удивительные эффекты. Тени могут быть однослойными или многослойными, размытыми, цветными или полупрозрачными.

Каждая тень применяется как к самому тексту, так и к элементам его оформления. Одновременно можно задавать несколько теней, указывая их через запятую. Тени накладываются друг на друга, не перекрывая сам текст. Первая тень всегда расположена сверху над остальными тенями. Свойство text-shadow наследуется.

Каждая тень определяется двумя или тремя значениями и цветом. Если цвет тени не указан, используется цвет текста.

text-shadow: x-offset y-offset blur color;
  • x-offset - смещение тени по горизонтали относительно текста. Может принимать как положительные, так и отрицательные значения. Обязательное значение.
  • y-offset - смещение тени по вертикали относительно текста. Может принимать как положительные, так и отрицательные значения. Обязательное значение.
  • blur - определяет радиус размытия тени. Чем больше радиус, тем сильнее размыта тень. Необязательное значение.
  • color - по умолчанию принимает цвет текста. Для значения можно использовать hex, rgb, rgba форматы записи цвета. Необязательное значение.

Если значения длин отрицательные, то тень смещается влево и вверх от текста.

box-shadow

Свойство box-shadow добавляет элементу одну или более теней. Тени бывают внешние или внутренние, размытые или плоские, они могут следовать контурам блоков со скругленными углами. С помощью ключевого слова inset создаются тени внутри элемента, делая элемент визуально объёмным или вдавленным.

Каждая тень принимает от одного до пяти параметров:

box-shadow: inset x-offset y-offset blur spread color;
  • x-offset - смещение тени по горизонтали относительно блока. Может принимать как положительные, так и отрицательные значения, положительное смещает тень вправо от блока, отрицательное – влево. Обязательное значение.
  • y-offset - смещение тени по вертикали относительно блока. Может принимать как положительные, так и отрицательные значения, положительное смещает тень вниз, отрицательное — вверх. Обязательное значение.
  • blur - определяет радиус размытия тени. Чем больше радиус, тем сильнее размыта тень. Можно использовать только положительные значения. Необязательное значение.
  • spread - расширяет тень, утолщая ее. Принимает как положительные, так и отрицательные значения. Необязательное значение.
  • color - Необязательное значение. По умолчанию тень черного цвета. Для установления значения можно использовать hex, rgb, rgba форматы записи цвета.
  • inset - создаёт тень внутри блока.

CSS-трансформации

CSS-трансформации позволяют изменять размер, форму и положение элемента на веб-странице с помощью свойства transform.

Трансформации преобразовывают элемент так, что другие элементы не сдвигаются относительно него. По умолчанию трансформация происходит относительно центра элемента.

Наиболее часто используют краткую форму записи. Краткая форма эквивалентна записи в скобках - где изменяется горизонтальная и вертикальная координаты.

Важно запомнить, что при последовательном применении этих значений смена последовательности может привести к другому результату. Причина - точка отсчета преобразования. Эта точка отсчета смещается при каждом преобразовании.

Translate

Перемещает элемент в горизонтальном и/или вертикальном направлениях.

translate (translateX, translateY), translate3d, translateZ

Rotate

Вращение вдоль оси X, Y или Z. Вращение вдоль оси X, Y выглядит как сжатие изображения, а вдоль Z - как вращения.

rotate(rotateX, rotateY), rotate3d, rotateZ

Scale

Изменение масштаба элемента. Может перекрывать отображение соседних элементов.

scale(scaleX, scaleY), scale3d, scaleZ

Skew

Используется для искажения сторон элемента относительно координатных осей. Если указано одно значение, второе будет определено браузером автоматически.

skew(skewX, skewY)

transform-origin

Позволяет сместить центр трансформации, относительно которого происходит изменение положения/размера/формы элемента. Значение по умолчанию — center, или 50% 50%. Задаётся только для трансформированных элементов. Не наследуется.

CSS-переходы

CSS-переходы позволяют анимировать исходное значение свойства на новое значение с течением времени, управляя скоростью смены значений свойств.

Смена свойств происходит при наступлении определенного события, которое описывается соответствующим псевдоклассом, например :hover.

Cвойство transition это краткая запись четырех свойств:

СвойствоОписание
transition-propertyСодержит название CSS-свойств, к которым будет применен эффект перехода. Значение свойства может содержать как одно свойство, так и список свойств через запятую.
transition-durationЗадаёт промежуток времени, в течение которого должен осуществляться переход. Если разные свойства имеют разные значения для перехода, они указываются через запятую. Если продолжительность перехода не указана, то анимация при смене значений свойств происходить не будет.
transition-timing-functionСвойство задаёт временную функцию, которая описывает скорость перехода объекта от одного значения к другому. Задается функцией Безье.
transition-delayЗадержка, после которой начнется анимация.
transition-property: color;
transition-duration: 1.2s;
transition-timing-function: cubic-bezier(.20, .96, .74, .07);
transition-delay: .5s;

Рассмотрим пример изменения нескольких свойств.

В transition-property для этих свойств заданы начальные значения. При срабатывании псевдокласса hover и active в перечислении transition-property указаны свойства, значения которых надо анимировать и их конечные значения. В обоих случаях длительность анимации составит 2 секунды.

Краткая запись перехода

Все свойства, отвечающие за изменение внешнего вида элемента, можно объединить в одно свойство transition.

transition: transition-property transition-duration transition-timing-function transition-delay;

Переходы применяются не ко всем свойствам и их значениям. С полным перечнем можно ознакомиться на MDN.

CSS-анимация

Как и переходы, анимация придаёт сайтам динамичность, но в отличие от переходов, анимации не нужен обязательный инициатор и между начальным и конечным состояними возможно много промежуточных состояний.

Создание анимации базируется на ключевых кадрах, которые позволяют автоматически воспроизводить и повторять эффекты на протяжении заданного времени, а также останавливать анимацию.

Анимация реализована свойством animation. Не все свойства (но большинство) элемента могут быть анимированы. Полный список свойств, которые анимировать можно, находится на MDN

Создание анимации

Создается анимация путем установки ключевых кадров - @keyframes, минимум два ключевых кадра. Кадры определяют, какие свойства на каком шаге будут анимированы. Каждый кадр может включать один или более блоков объявления из одного или более пар свойств и значений.

Ключевые кадры создаются с помощью ключевых слов from и to ( 0% и 100% ) или с помощью процентов.

/* имя должно быть описательным */
  @keyframes animationName {
    from {
      /* Здесь перечисляются CSS-свойства */
      opacity: 0;
    }
    to {
      /* Здесь перечисляются CSS-свойства */
      opacity: 1;
    }
  }

Применение анимации

  • Если просто добавить анимацию к стилю, который тут же применяется к элементу, то анимация будет применена при загрузке страницы.
  • Анимацию можно применить к одному из псевдоклассов, включая :hover , :active , или :focus.
  • Анимацию можно применить к стилю класса, и воспользоваться JavaScript.
  • К одному и тому же элементу можно применить более одной анимации, записав их через пробел.
  • Виды анимаций можно перечислять через запятую.
@keyframes animationName {
  from { opacity: 0; }
  to { opacity: 1; }
}

.className {
  animation-name: animationName;
  animation-duration: 1s;
}

Краткая запись свойства animation

Значения свойств нужно перечислять в таком порядке

  1. animation-name - имя анимации, рекомендуется использовать название, отражающее суть анимации. Обязательно указывать.
  2. animation-duration - устанавливает продолжительность анимации, задаётся в секундах или миллисекундах, отрицательные значения не допустимы. Обязательно указывать.
  3. animation-timing-function - определяет изменение скорости от начала до конца анимации с помощью временных функций.
  4. animation-iteration-count - позволяет запустить анимацию несколько раз.
  5. animation-direction - задает направление повтора анимации.
  6. animation-delay - задает задержку анимации.
  7. animation-fill-mode - определяет порядок применения определенных в @keyframes стилей к объекту.
animation: animation-name animation-duration animation-timing-function animation-delay animation-iteration-count animation-direction;
.fade {
  animation: fadeOut 2s ease-in-out 2 alternate 5s forwards;
}

/* Соответсвует */

.fade {
  animation-name: fadeOut;
  animation-duration: 2s;
  animation-timing-function: ease-in-out;
  animation-iteration-count: 2;
  animation-direction: alternate;
  animation-delay: 5s;
  animation-fill-mode: forwards;
}

Пауза анимации

.fade:hover {
  animation-play-state: paused;
}
