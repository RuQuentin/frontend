Позиционирование элементов

Браузер отображает на странице содержимое документа html, следуя потоку выполнения документа. Для европейских языков поток выполнения такой же, как правила чтения - сверху вниз, слева направо.

А как будет, если элемент расположен над границей двух блоков? Или как заставить меню оставаться всегда на одном месте страницы, в то время как остальное содержимое можно перемещать с помощью скролла? Именно для решения подобных задач было создано свойство position.

Это свойство может словно приподнять элемент над потоком выполнения и разместить элемент над любым элементом, находящимся в потоке. А что будет если два таких элемента вне потока окажутся в одном месте, кто будет в тени, а кто на вершине? Эту проблему решает другое свойство - z-index.

position

Задает особые правила размещения элемента. В частности для определения положения элемента используются свойства top, left, bottom или right, которые неприменимы к элементам без position.

Относительное позиционирование

При position: relative, элемент остается в потоке: его размеры и положение рассчитываются для всех соседних блоков согласно box-model. Прорисовка изображения производится браузером с учетом заданный свойств top, left, bottom или right. При этом может происходить перекрытие изображения соседних элементов.

Смещение элемента R с position: relative

Запомнить, как применять свойства top, left, bottom или right для position: relative довольно легко. Перечисленные свойства указывают на то, куда будет визуально перемещен левый верхний угол нового (видимого) положения:

  • left: 20px - на 20px левее от положения в потоке
  • right: 20px - на 20px правее от положения в потоке

Абсолютное позиционирование

position: absolute - поднимает элемент над потоком, т.е. после применения этого свойства соседние элементы исключат этот элемент из расчета для определения своего положения.

Отсчет своих top, left, bottom или right для absolute-элемента ведется относительно ближайшего предка со свойством, отличным от свойства position: static. Если такового предка не окажется, то отсчет будет произведен от body.

Когда к элементу добавляется свойство position: absolute, то всегда следует ответить на вопрос: "Какой элемент будет являться тем предком, к которому потребуется применить свойство position: relative ?". И после ответа на этот вопрос необходимо убедиться, что position: relative установлено.

Если при расчете положения элемента в потоке нужно было учитывать только верхний левый угол элемента и родительского контейнера, то отсчет положения абсолютно позиционированного элемента другой.

Запись bottom: 20px означает расстояние между нижней границей родительского контейнера и нижней границей самого элемента. Аналогично работают и right, left и top - задается расстояние между одноименными сторонами контейнера и элемента.

  • right: 50px - правая граница элемента на 50px от правого края контейнера
  • top: 50px - верхняя граница элемента на 50px от верхнего края контейнера
  • bottom: 50px - нижняя граница элемента на 50px от нижнего края контейнера
  • left: 50px - левая граница элемента на 50px от левого края контейнера

Ниже - скриншот, демонстрирующий применение position: absolute. Черная точка указывает на начало отсчета, т.е. верхнюю левую точку первого родительского контейнера с position: relative. Блок по центру отцентрирован с помощью absolute и transform: translate. Остальные блоки показывают смещение в зависимости от свойств и значений.

Свойство transform и его значение translate(translateX, translateY) смещают элемент от его текущего положения. translateX смещает верхний левый угол элемента по горизонтали, а translateY - по вертикали. Т.е. расчет положения такой же, как и в box-model. Такое дублирование способа перемещения нужно потому, что с помощью свойства transform можно осуществлять анимацию, о чем подробно будет рассказано позже.

Не стоит забывать, что на элементы с position продолжают действовать правила, по которым эти элементы будут смещаться при margin и менять размер при padding. Эта особенность позволяет центрировать элемент по вертикали контейнера.

Для этого смещаем левый верхний край по центру контейнера, а затем смещаем весь элемент влево и вверх на половину ширины и высоты самого элемента.

element {
  position: absolute;
  left: 50%;
  right: 50%;
  transform: translateX(-50%) translateY(-50%);
}

Позиционирование с фиксацией положения

position:fixed это аналог значения absolute. Отличие заключается в точке отсчета. Для fixed отсчет всегда ведется от границ окна браузера. Это создает эффект фиксации элемента при использовании вертикальной прокрутки. Как правило, свойство position: fixed используют для элементов навигации, позволяющих перейти к началу страницы, или для верхнего меню, как в примере на изображении.

для демонстрации работы position fixed наведите курсор под надписью "I am fixed" и используйте колесо мыши для прокрутки контента, т.к. scrollbar этого примера был искусственно спрятан.

"Липкое" позиционирование

position: sticky это комбинация свойств relative и fixed. Пока элемент находится в своем контейнере и контейнер не покидает область экрана при вертикальной прокрутке это relative. Если часть родительского контейнера покинула видимую часть экрана, то свойство меняется на fixed, т.е. фиксируется на странице пока часть контейнера присутствует на экране. Если весь контейнер покинет область видимости, то опять включится свойство relative, т.е. изображение элемента исчезнет с экрана вслед за контейнером.

Отмена действия позиционирования

position: static это обычное (по умолчанию) значение свойства position всех элементов. Его можно изменить, задав другое значение. А затем отменить действие relative, absolute или fixed присвоив снова значение static.

Свойство z-index

До свойства position мы не придавали большого значения расположению элементов по оси z (перпендикуляру к плоскости экрана). Расположение элементов в этой плоскости было интуитивно понятно. Предок располагался внизу, подобно листу бумаги, а потомок - словно новый лист бумаги, который ложится сверху. И чем больше потомков было у элемента, тем больше было таких слоев.

Но применение position все изменило. В отличие от элементов float, которые выпадают из потока, но не перекрывают друг друга, элементы с position, отличном от static, ведут себя по другому. Элементы выпадают из потока, а значит для таких элементов нужно правило, которое будет определять какой элемент находится выше, а какой ниже. Это правило задает свойство z-index, значениями которого может быть любое целое число, в том числе и отрицательное.

Правила применения

Общий предок создает контекст, область в которой z-index элементов оказывает влияние на другие элементы с z-index. Внутри этого контекста значение z-index может быть больше других и тогда элемент располагается сверху.

Минимальное (даже отрицательное) значение z-index говорит о том, что другие элементы вне потока будут выше этого элемента. И в случае равенства значения z-index (это значение по умолчанию) выше будет элемент объявленный последним.

Исключением из правил может стать ситуация, когда какой-то из предков без свойства z-index получит полупрозрачный фон, т.е. его opacity станет отличной от 1. В таком случае происходит формирование нового контекста и подобных случаев лучше избегать в виду неочевидности применения правил из-за неожиданного появления нового общего предка.

CSS-Tricks: position

Верстка выпадающего меню

Выпадающее меню - это наиболее популярный тип меню на веб-страницах. Рассмотрим этапы создания такого меню. Предложенные имена классов приведены для ясности изложения.

  1. Необходимо создать обычное меню, присвоив class="menu" блоку и class="menu-item" элементам меню.
  2. Внутри menu-item разместите новое меню, с class="submenu" и элементами class="submenu-item".
  3. Скрываем вложенное меню
    .submenu { display: none; }
  4. Задаем правило появления вложенного меню при наведении курсора
    .menu-item:hover > .submenu { display: block; }
  5. С помощью position: absolute размещаем submenu относительно li так, чтобы была возможность перейти к пунктам submenu.

Т.е. ключевым является правило появления вложенного меню. Это правило работает так: при наведении на элемент меню, содержащий подменю, сработает псевдокласс hover и прямой потомок (на это указывает знак > ) станет видимым.

Также крайне важен последний этап. Например, при наличии 1px границы (border) у menu может происходить отмена действия hover. Дело в том, что на сам блок menu эффект hover не распространяется. Его действие связано только с menu-item и если браузер зафиксирует курсор не над menu-item, то вернется состояние по умолчанию и подменю исчезнет. Такую ошибку сложно обнаружить, т.к. браузер не отслеживает каждый пройденный курсором пиксель, и ошибка проявляется время от времени.

Стандарт HTML5

Cемантическая верстка предназначена для описания смысловой структуры документа. А именно для разделения контента на различные по типу и содержанию части: заголовки, абзацы, изображения, ссылки и т.п.

До HTML5 часть тегов несла минимальную смысловую нагрузку "контейнер" - div и span. А часть тегов вообще имела отношение не к содержанию, а к внешнему виду - br, hr, i, b и т.п. HTML5 предоставил новые элементы для структурирования и группировки контента, способных передать семантику блока. Например, header и footer.

С практической точки зрения подобные изменения облегчают анализ содержимого веб-страниц поисковыми роботами, программам для людей с ограниченными возможностями и т.п.

Всего выделяют 7 групп элементов, по смысловому содержанию:

  1. Мета содержимое
  2. Потоковое содержимое
  3. Секционное содержимое
  4. Заголовочное содержимое
  5. Текстовое содержимое
  6. Встроенное содержимое
  7. Интерактивное содержимое

Структурные элементы

В то время как div может использоваться для представления нижеперечисленных разделов, использование семантичных элементов обеспечивает гораздо более богатый и значимый документ.

header

Заголовок страницы или раздела, содержащий вводный контент или навигацию. Не является обязательным.В HTML-документе может содержаться одновременно несколько элементов header и они могут располагаться в любой части страницы.

header нельзя помещать внутрь элементов: footer, address, header

nav

Предназначен для создания блока навигации веб-страницы или всего веб-сайта. Блок навигации - основная группа ссылок на другие страницы или разделы страницы. На странице может быть несколько элементов. Не заменяет теги ul или ol, он просто их обрамляет, тем самым придавая семантическое значение.

Не все группы ссылок должны быть обернуты nav, этот элемент предназначен для разделов, которые состоят из главных навигационных блоков.

article

Раздел, целиком содержащий некоторую тему - "независимая часть документа или сайта". Используется для группировки записей — публикаций, статей, записей блога. Как правило, начинается с заголовка.

section

общий раздел документа. Может использоваться для разделения статьи или, например, для представления глав. Это универсальный раздел документа. Группирует тематическое содержимое и обычно содержит заголовок.

Не является блоком-оберткой, для этих целей используйте элемент div.

main

основное содержание body - уникальное для страницы, исключая контент, который может повторяться на нескольких страницах, например, навигация.

Не может быть вложен в элементы: article, aside, footer, header или nav

aside

Группирует содержимое, связанное с окружающим его контентом. Раздел, располагающийся сбоку от основного, содержит менее значимую информацию (рекламные блоки, архив и т.п.)

Не подходит для блоков, просто позиционированных в стороне.

footer

Нижний колонтитул страницы или раздела. Может содержать контент, например, постскриптум, приложение или ссылки на связанные страницы, данные о копирайте, контактную информацию.

В документе может быть несколько элементов footer. Как каждая страница, так и каждая статья может иметь свой footer.

figure и figcaption

Автономный иллюстративный контент, который обычно упоминается в основном содержании страницы. Используется для добавления описани к фотографиям, диаграммам, фрагментам кода. figcaption — потомок figure. Используется как контейнер для содержания описательной части изображения.

Элементы семантики текста

  • address - для отображения контактных данных владельца документа или статьи
  • time - определяет время или дату. Для тега доступен атрибут datetime, его значение будет видеть пользователь на экране своего компьютера.
  • mark - выделяет фрагменты текста, помечая их желтым фоном, обычно при совпадении с текстом в строке поиска.
  • small - комментарии - отображает текст шрифтом меньшего размера
  • cite - для указания источника цитирования.
  • wbr - возможное место разрыва длинной строки
  • bdi, bdo - для текста, написанного на языке с потоком отличным от потока документа
  • ruby, rb, rt, rtc, rp - для работы с восточными языками (иероглифами)
An Overview of HTML5 Semantics

HTML5 media

В HTML5 использование потокового медиаконтента стало стандартным и не требует использования сторонних инструментов. Рассмотрим как работать с потоковым видео и звуком.

audio

Парный тег, атрибуты которого определяют все его возможности:

  • src - определяет источник звука или набор источников в разных форматах
  • controls - элементы управления прослушиванием, определяемые браузером
  • loop - если этот атрибут задан, то воспроизведение будет повторяться снова и снова
  • preload - задает способ кеширования:
    • none: не загружает ничего до запуска пользователем.
    • metadata: загружает только название трека, продолжительность и т.п. данные, но не звук.
    • auto: загружает все.

Кроме того, тег source может быть вложен в тег audio и содержать атрибуты src для потоков с разными кодеками.

Поддержка кодеков ogg, mp3, wav. Если браузер не поддерживает предлагаемые вами кодеки, то между тегами audio принято размещать текст-предупреждение.

<audio controls>
  Your browser does not support the audio element.
  <source src="boom.wav" type="audio/wav">
  <source src="boom.ogg" type="audio/ogg">
</audio>

video

Парный тег, имеющий аналогичные с audio атрибуты, а также дополнительные:

  1. width и height задают размеры для отображения видеопотока
  2. autoplay автоматический запуск проигрывания
  3. poster ссылка на изображение, которое станет фоном выделенного width и height пространства до запуска видеопотока

Поддержка кодеков ogv, mp4, flv, webm

Кроме того, тег track может быть вложен в тег video и содержать атрибуты для субтитров kind="subtitles" src="boom.en.vtt" srclang="en" label="English" или другие данные. Этот тег также может использоваться для audio.

<video controls poster="./images/sample.gif">
  Your browser does not support the video element.
  <source src="boom.mp4" type="video/mp4">
  <source src="boom.ogv" type="video/ogv">
  <track kind="subtitles" src="boom.en.vtt" srclang="en" label="English">
</video>

Переменные CSS

Переменные в CSS на данный момент поддерживаются всеми основными браузерами за исключением частичной поддержки Edge и не поддерживаются IE и Opera Mini. Тем не менее, эта технология обязательно будет использоваться в самое ближайшее время и стоит рассказать о ее возможностях.

В отличие от переменных в препроцессорах CSS (рассказ о них будет позже), переменные CSS остаются переменными и во время выполнения программы, в то время как переменные препроцессоров после компиляции в CSS становятся обычными значениями. Этот факт очень удобен для создания JavaScript приложений.

Довольно просто создавать интерактивность при оформлении страницы (пользователь может изменять любые параметры CSS, если предоставить ему такую возможность). Но основное применение переменных в CSS это сокращение объема кода и упрощение его поддержки.

Синтаксис

Для объявления переменной перед свойством достаточно указать два дефиса, а для использования вместо значения написать var() и внутри круглых скобок поместить имя переменной с двумя дефисами впереди.

element { --main-bg-color: teal; }

element { background-color: var(--main-bg-color); }

Проблема в том, что объявленная таким образом переменная будет видна только внутри селектора, где произошло объявление переменной. Для того чтобы переменную можно было использовать в любом месте программы (объявить переменную глобально) необходимо объявить ее внутри специального селектора :root.

:root { --main-bg-color: teal; }

.one { background-color: var(--main-bg-color); }

.two { background-color: var(--main-bg-color); }

.three { background-color: var(--main-bg-color); }
