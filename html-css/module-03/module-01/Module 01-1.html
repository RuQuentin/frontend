Введение

Когда мы говорим о программировании, первое что приходит в голову это набор инструкций в файл. Содержимое этого файла называется исходный код.

Исходный код (source code) - набор фраз, слов, специальных символов и т.д., специфичных для данного языка программирования. Все эти конструкции созданы для того, чтобы сказать машине что необходимо сделать, это и есть синтаксис языка.

Но слова, фразы, символы и т.д., которые составляют программу, на самом деле непонятны для машины. Поэтому есть шаг, выполняющийся после того как вы написали программу, который конвертирует синтаксис языка в файле, в набор инструкций понятных компьютеру. Этим занимается специальная программа: компилятор или интерпретатор, которые преобразуют исходный код в набор нулей и единиц, так как компьютер не знает ничего кроме 0 и 1.

Отсюда можно сделать вывод, исходный код написаный человеком на самом деле пишется не для машины. Сам по себе исходный код это побочный артефакт того, что необходимо машине для выполнения кода.

Получается можно написать много разных вариантов исполнения одной и той же программы, а компилятор выдаст машине одинаковый набор нулей и единиц. Так какое имеет значение как мы пишем исходный код?

Исхоный код пишется не для машины, да компилятор использует его для составления машинного кода, но для разработчика, для человека. Исходный код пишется для вас самих, чтобы спустя какое-то время вы могли посмотреть в код и понять что там написано, как исполняется программа, какая у нее логика и т.д. А так же для других разработчиков, которые будут пользоваться вашим кодом.

Именно поэтому важно писать код так, чтобы он не только верно решал задачу, но и имел смысл когда на него смотришь. Код должен быть понятным, легко читаемым. Это одна из самых сложных задач начинающих (и не только), на ее решение уйдет много времени и сил.

JavaScript

JavaScript - это язык сценариев, предназначенный для взаимодействия с элементами веб-страниц и добавления интерактивности. При Front-end разработке, JavaScript используется с HTML и CSS для обеспечения функциональности веб-страницы, такой как создание интерактивных карт, отображение анимированных графиков и т. д.

Когда загружается веб-страница, то есть после загрузки HTML и CSS, JavaScript-движок веб-браузера начинает выполнять код JavaScript. Затем код JavaScript модифицирует HTML и CSS для динамического обновления пользовательского интерфейса.

JavaScript состоит из трех основных частей:

  • ECMAScript - встраиваемый расширяемый не имеющий средств ввода-вывода стандарт языка программирования, используемый в качестве основы для построения других скриптовых языков. Обеспечивает основные функции.
  • Объектная модель документа (DOM) - предоставляет интерфейсы для взаимодействия с элементами на веб-страницах.
  • Модель объекта браузера (BOM) - предоставляет интерфейсы для взаимодействия с веб-браузерами. Позволяет получать такие данные как местонахожнение пользователя, версия и тип браузера в котором открыли веб-страницу и т.д.

Когда JavaScript используется на веб-странице, он выполняется в веб-браузере пользователя, поэтому мы называем JavaScript клиентским (client-side). Но JavaScript может работать как в веб-браузерах, так и на серверах. Популярной серверной средой для JavaScript является NodeJS.

Логические составляющие

Когда мы пишем код, важно не просто знать какой символ или конструкцию можно использовать, но в первую очередь мы должны понимать терминологию и составляющие нашего исходного кода.

Statements

Инструкция(statement) - это связанный набор слов и символов из синтаксиса языка, которые объединяются, чтобы выразить одну идею, одну инструкцию для машины.

a = b * 2;

Это пример инструкции. В JavaScript можно (обычно) различить инструкцию по точке с запятой в конце. Точку с запятой в данном случае можно сравнить с точкой в конце предложения вашего родного языка, она символизирует о конце предложения.

  • a и b называются переменными, это такие коробки в которых мы можем хранить информацию. Переменные содержат значения, которые использует программа. Думайте о них как о символических представителях значений.
  • 2 - просто значение. Это называется значением литерала (literal value), так как оно не хранится в переменной.
  • = и * - операторы, они производят действия над значениями и переменными.

Представим что переменная b уже содержит значение 21. Тогда наша инструкция говорит машине:

  • Пойди найди переменную b и спроси какое у нее сейчас значение.
  • Подставь значение переменной b в утверждение на место b.
  • Умнож 21 на 2, получив 42.
  • После чего, запиши результат вычисления выражения правой части (то что справа от оператора равно) в переменную a.

Expressions

Инструкции состоят из частей, как в любом языке утверждения состоят из фраз, и эти фразы называются выражениями.

Выражение (expression) - ссылка на переменную или значение, или на набор переменных и значений в сочетании с операторами.

[ [a] = [ [b] * [2] ] ];

Инструкция из примера выше содержит 5 выражений (выделены квадратными скобками):

  • [2] - выражение значения литерала.
  • [b] и [a] - выражения переменной, означают необходимость подставить значение переменной.
  • [b * 2] - арифметическое выражение. В данном случае умножение.
  • [a = b * 2] - выражение присваивания. В данном случае указывает на необходимость вычисления правой части выражения и присваивания результата переменной a в левой части выражения.

Так же есть выражения вызова, выражения сравнения и т. д. Мы не будем сейчас рассматривать их все, нам важно понимать из каких частей состоит исходный код и как правильно его читать.

Методы подключения JavaScript к странице

Чтобы добавить JavaScript к странице, используется тег script в HTML файле. Существует два способа использования элемента script на странице.

Вставить JavaScript в HTML документ

Для вставки скрипта в документ мы просто пишем <script>, без дополнительных атрибутов и комментариев. Когда браузер видит тег <script> без дополнительных атрибутов, он начинает исполнять JavaScript который находится внутри тега.

Размещение JavaScript внутри элемента <script> напрямую не рекомендуется и должно использоваться только для доказательства концепции или цели тестирования.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Module-1</title>
  </head>

  <body>
    <script>
      alert('Let\'s learn JS!');
    </script>
  </body>
</html>

Внешний файл JavaScript

Указать в HTML документе ссылку на внешний JavaScript файл. Чтобы включить JavaScript из внешнего файла:

  1. Сначала создайте файл с расширением .js и поместите его в подпапку js. Размещение JavaScript файла в папке js не требуется, однако это хорошая практика.
  2. Затем используйте URL-адрес в исходном коде JavasScript в атрибуте src элемента script.
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Module-1</title>
  </head>

  <body>
    <!-- контент -->
    <script src="js/scripts.js"></script>
  </body>
</html>

При отображении HTML документа, некоторое (не продолжительное) время может отображаться пустая страница, дело в том что фаза рендеринга страницы преостаналивается до тех пор, пока не будет полностью загружен скрипт (если он указан в head). Чтобы этого избежать, включать файл JavaScript стоит в конце тега body после содержимого, как показано в примере.

Множественные файлы скриптов

Когда вы включаете несколько JavaScript-файлов на страницу, механизм JavaScript интерпретирует файлы в том порядке, в котором они указаны в HTML файле. Браузер загружает и отображает HTML постепенно. Если браузер видит тег script, то он по стандарту обязан сначала выполнить его, а потом показать оставшуюся часть страницы. Такое поведение называют «синхронным».

Механизм JavaScript будет последовательно интерпретировать файлы script-1.js и script-2.js. Он завершает интерпретацию файла script-1.js перед интерпретацией файла script-2.js.

<script src="/path/to/script-1.js"></script>
<script src="/path/to/script-2.js"></script>

Атрибуты async и defer

Чтобы изменить порядок загрузки и выполнение кода JavaScript, можно использовать один из двух атрибутов элемента script - async и defer. Эти атрибуты применяются только к внешнему файлу скрипта.

async

Указывает браузеру о том что он должен как можно скорее выполнить асинхронный файл JavaScript. Атрибут async не гарантирует выполнения файлов скриптов в указанном порядке. Файл script-2.js может выполняться перед файлом script-1.js, поэтому вы должны убедиться, что между ними нет зависимости.

defer

Просит веб-браузер выполнить файл сценария после полного анализа и загрузки документа. Даже если мы поместим элемент script в раздел head, сценарий будет ожидать пока браузер обработает закрывающий тег html, чтобы начать выполнение. Браузер гарантирует, что относительный порядок скриптов с defer будет сохранён. Скрипт с defer сработает, когда весь HTML-документ будет обработан браузером.

Директива use strict

Изначально язык JavaScript развивался без потери совместимости. Новые возможности добавлялись в язык, но старые – никогда не менялись, чтобы не «сломать» уже существующие веб-страницы с их использованием. Однако, это привело к тому, что любая ошибка в дизайне языка становилась «вшитой» в него навсегда. Так было до появления стандарта ECMAScript 5, который одновременно добавил новые возможности и внёс в язык ряд исправлений, которые могут привести к тому, что старый код, который был написан до его появления, перестанет работать.

Строгий режим - это новая возможность в спецификации ECMAScript 5, которая позволяет переводить скрипт в режим полного соответствия современному стандарту. Это предотвращает определенные действия, такие как использование небезопасных конструкций. Для того чтобы перевести скрипт в строгий режим, достаточно просто указать директиву "use strict" в начале js файла.

Синтаксис JavaScript

JavaScript чувствителен к регистру

Все в JavaScript, включая переменные, имена функций, имена классов и операторы, чувствительны к регистру. Это означает, что переменные user, usEr и User различны.

Идентификаторы

Идентификатор - это имя переменной, функции, параметра или класса. Идентификатор состоит из одного или нескольких символов в следующем формате:

  • Первым символом должна быть буква a-z или A-Z, символ подчеркивания _ или знак доллара $.
  • Другие символы могут быть буквами a-z, A-Z, цифрами 0-9, подчеркиваниями _ и знаками доллара $.

Стандартной практикой является использование camelCase для идентификаторов, при котором первое слово полностью пишется строчными буквами, а каждое дополнительное слово начинается с прописной буквы.

  • user
  • greetUser
  • getUserData
  • isActive

Комментарии

JavaScript поддерживает однострочные и блочные комментарии. Однострочный комментарий начинается с двух символов прямой косой черты //, например:

// this is a single-line comment

Блочный комментарий начинается с косой черты и звездочки /* и заканчивается противоположным */ в следующем примере.

/*
* This is a block comment that can
* span multiple lines
*/

Точка с запятой

Хотя JavaScript не требует завершения инструкции точкой с запятой - ;. Рекомендуется всегда использовать точку с запятой для завершения блока инструкции.

Эта практика сделает ваш код более читабельным и поможет избежать многих ошибок, с которыми вы можете столкнуться. Кроме того, в будущем мы научимся сшивать и сжимать JavaScript код, прежде чем разворачивать его в рабочей среде, чтобы удалить лишнее пространство для уменьшения размера файла. Если сделать это без точек с запятой, будут синтаксические ошибки.

Переменные

Переменные используются для хранения данных. Думайте о них как о коробках с названием, в которые мы можем положить какие-то данные. Переменная является только именованным заполнителем для значения, она состоит из имени и выделенной области памяти, которая ему соответствует.

Имена переменных

  • Имя может состоять из: букв, цифр, символов $ и _
  • Первый символ не должен быть цифрой
  • Регистр букв имеет значение: apple и aPplE это две разные переменные
  • Зарезервированные имена описаны в таблице выше
  • Имя переменной должно быть понятным
ПлохоХорошо
yabloko, chislo, massivProductov, spisok_iz_4apple, number, goods, list

var, let, const

Существует целых 3 ключевых слова для объявления переменных.

  • var - сокращение от variable, использовалось в предыдущих версиях языка, можно применять и сейчас.
  • let - современная замена var. В переменную объявленную через let можно присвоить новое значение.
  • const - "константа", используется как ключевое слово let, но переменная, которую вы объявляете, должна быть немедленно инициализирована значением, и этой переменной не может быть присвоено другое значение.

Используйте const если вы знаете что в переменную не нужно будет записать новое значение. Используйте let если в переменную нужно будет записать новое значение.

Не создавайте переменные без объявления (т.е. без let или const). Такие переменные могут стать глобальной, т.е. доступной в общей области видимости всего кода. Что может вызывать непредвиденное поведение программы и создать трудно обнаруживаемую ошибку.

undefined vs. undeclared

Важно различать неопределенные и необъявленные переменные.

  • Неопределенная переменная (undefined) - это переменная, которая была объявлена, но поскольку мы не присвоили ей значение, переменной присвоилось значение undefined как ее начальное значение.
  • Необъявленная переменная (undeclared) - та которая не была объявлена в доступной области видимости.
let declaredVariable;
//Переменная declaredVariable объявлена, но не инициализирована,
// поэтому ее значение undefined (не определено),
// тогда как переменная undeclaredVariable не была объявлена,
// поэтому доступ к ней вызывает ссылку ReferenceError

console.log(
  declaredVariable
); // undefined

console.log(
  undeclaredVariable
); // ReferenceError: undeclaredVariable is not defined

Поднятие перменных

Hoisting (Поднятие перменных) - это "механизм", при котором "движок" JavaScript перемещает все объявления переменных в верхнюю часть области видимости, будь то функция или глобальная область.

Hoisting, на самом деле, это не существующий механизм языка, метафора, которую мы используем для описания того, как переменные привязываются к их лексическим средам во время компиляции.

Если вы объявите переменную с ключевым словом var, переменная будет поднята до начала(верха) охватывающего ее лексического окружения. В результате, если вы обращаетесь к такой переменной перед ее объявлением, переменная оценивается как неопределенная undefined, так как во время исполнения кода ей изначльно будет присвоено это значение.

// То как вы пишите код
console.log(a); // undefined , почему?
var a = 5;
console.log(a); // 5

// То как на самом деле "выглядит" код после фазы компиляции
var a = undefined;

console.log(a); // undefined , почему? - потому что объявление
// переменной было "поднято" в начало области видимости и
// инициализировано undefined

a = 5;
console.log(a); // 5

Если вы объявите переменную с ключевым словом let или const, в отличии от многих ошибочных заявлений, переменная так же будет поднята до вершины охватывающего ее лексического окружения. Но во время исполнения кода, при входе в область видимости где она была объявлена, она не будет инициализирована значением undefined до той строки кода, где она изначально была объявлена ключевым словом let или const. В результате, если вы обращаетесь к такой переменной перед ее объявлением, результатом будет ReferenceError. Это тот механизм, что в спецификации описывается как TDZ (temporal dead zone).

// ES5
console.log(a); // undefined
var a = 5;
console.log(a);// 5

// ES6
// Вот пример еффекта TDZ, любая попытка вызвать
// пременную по имени до ее определения в коде вызовет ошибку
// console.log(b); // ReferenceError: b is not defined
let b = 10;
console.log(b); // 10

// console.log(c); // ReferenceError: c is not defined
const c = 15;
console.log(c); //15

Типы примитивов

JavaScript это динамический, слабо типизированный язык, поэтому сама переменная не ассоциируется с каким-либо типом, только ее значение. Тип данных описывает, какие данные хранятся. Другими словами, одна и та же переменная может хранить значения разных типов в любое время.

JavaScript имеет шесть примитивных типов данных. Следующие типы данных называются примитивными, заключается в том, что они считаются базовыми строительными блоками языка программирования.

Грамматика и типы

Number

Целые числа и числа с плавающей запятой. После того, как вы объявите переменную, вы можете инициализировать ее числовым значением. Более детально описаны в разделе Числа .

let age = 20;
let number = 5.8;

String

Строки в JavaScript - это просто текст, последовательность из нуля или более символов. Строка начинается и заканчивается либо одиночной ', либо двойными кавычками ".

Ключевым моментом для запоминания является то, что строка, начинающаяся с двойной кавычки, должна заканчиваться двойной кавычкой, а строка, начинающаяся с одиночной кавычки, должна заканчиваться одиночной кавычкой.

let name = 'Mango';

Более детально описаны в разделе Строки

Boolean

Логический тип данных имеет только два значения: true или false, в нижнем регистре.

СинтаксисЗначение
trueимеет смысл «да», «верно», «истина», 1
falseозначает «нет», «неверно», «ложь», 0

Обратите внимание на имена переменных содержащих булевое значение. Они задают вопрос, и ответ на его - да или нет.

let isAuthenticated = true;
let canMerge = false;
let hasChildren = true;

Null

Особое значение в JavaScript, которое по сути значит "ничто". Имеет только одно значение - null. Javascript определяет, что null - пустой указатель на объект.

Переменная apples пуста и лишена значения.

let apples = null;

Undefined

Еще одно специальное значение в JavaScript. Имеет одно значение undefined. По умолчанию, когда переменная объявляется, но не инициализируется, ее значение не определено, ей присваивается значение undefined, потому что значение не было присвоено этой переменной..

let apples;
console.log( apples ); // undefined

Symbol

Символы - это жетоны, которые служат уникальными идентификаторами.

Например, вы можете использовать их для определения значений констант, которые вы определяете. Предположим, вы управляете компанией, которая делает рубашки. Вы можете использовать символы для отслеживания всех продаваемых вами размеров:

const SIZE_SMALL = Symbol();
const SIZE_MEDIUM = Symbol();
const SIZE_LARGE = Symbol();

В реальном мире каждый размер отличается, и также будет каждое значение в этих константах. Конечно, вы можете сделать это без использования символов:

const SIZE_SMALL = "SMALL";
const SIZE_MEDIUM = "MEDIUM";
const SIZE_LARGE = "LARGE";

Но проблема в том, что ни одно из значений этих констант не может быть гарантировано уникальным. Ничто не помешало бы кому-либо просто создать новую строковую переменную и присвоить ей строку "SMALL".

Оператор typeof

Чтобы получить текущий тип значения переменной, используйте оператор typeof.

// Undefined

console.log(typeof a); // undefined

// Null
// object это не примитивный тип данных,
// о нем мы узнаем в следующих модулях

// Number

// Чтобы представить число с плавающей запятой,

// String

console.log(typeof message); // string

// Boolean

console.log(typeof isActive); // boolean

let isHappy = true;
console.log(typeof isHappy); // boolean

Преобразование типов примитивов

Система преобразования типов в JavaScript довольно проста, но отличается от других языков. Всего есть 3 типа преобразования:

  • Приведение к строке
  • Приведение к числу
  • Приведение к логическому типу

Приведение к строке

Строковое преобразование происходит, когда требуется представление чего-либо в виде строки. Тоесть в строковом контексте или при сложении со строкой. Работает очевидным образом, как есть: false становится "false", null – "null" и т.п.

Например, его производит функция alert.

let isActive = false;
  alert( isActive ); // "false"

Можно также осуществить преобразование явным вызовом String(val).

console.log( String(false) === "false" ); // true

Также для явного преобразования применяется оператор +, у которого один из аргументов строка. В этом случае он приводит к строке и другой аргумент.

console.log( true + "text" ); // "truetext"
  console.log( "100" + undefined ); // "100undefined"

Приведение к числу

Происходит в численном контексте, включая унарный плюс +value, в математических функциях и выражениях, а также при сравнении данных различных типов (кроме сравнений ===, !==).

Для преобразования к числу в явном виде можно вызвать Number(val), либо, что короче, поставить перед выражением унарный плюс +. Унарный плюс может не считаться лучшей практикой, так как усложняет чтение кода.

let a = +"100"; // 100
  let b = Number("100"); // 100

Полноценное правило преобразования строки таково: пробельные символы по краям обрезаются. Далее, если остаётся пустая строка, то она преобразовывается в число 0, иначе из непустой строки "считывается" число, при ошибке результат NaN.

ЗначениеПреобразуется в...
undefinedNaN
null0
true/false1/0
console.log(+"100text"); // NaN
  console.log(+undefined); // NaN
  console.log(+null); // 0
  console.log(+true); // 1
  console.log(+false); // 0
  console.log( "1" == true); // true
  console.log( "1" === true); // false

Интуитивно, значения null/undefined ассоциируются с нулём, но при преобразованиях ведут себя иначе. Для более очевидной работы кода и во избежание ошибок лучше не давать специальным значениям null/undefined участвовать в сравнениях > >= < <=.

Приведение к логическому типу

Преобразование к true/false происходит в логическом контексте, таком как if(value), и при применении логических операторов.

Очень просто запомнить что приводится к false во время логического приведения типов: 0, -0, NaN, null, undefined, пустая строка: "" или '', false. Все остальное приводится к true.

Числа

Все числа в JavaScript, как целые так и дробные, имеют тип Number и хранятся в 64-битном формате. В JavaScript можно записывать числа не только в десятичной системе счисления.

// Шестнадцатиричная система, начинается с 0x
console.log( 0xFF ); // 255

// Научный формат, число e количество нулей
console.log( 2e3 ); // 2000

// Если количество нулей отрицательно,
// то число сдвигается вправо за десятичную точку,
// так что получается десятичная дробь
console.log( 2e-3 ); // 0.002

В ES6, Number получил большое количество улучшений. Материал этого курса, руководствуясь лучшими практиками, использует именно улучшеные методы работы с числами. Отличную статью по Number обязательно прочитать тут .

Преобразование к числу

Большинство арифметических операций и математических функций преобразуют значение в число автоматически. Для того чтобы сделать это явно, обычно перед значением ставят унарный плюс +. При этом, если строка не является в точности числом, то результат будет NaN (Not a Number). Аналогичным образом происходит преобразование и в других математических операторах и функциях.

parseInt и parseFloat

В HTML/CSS многие значения не являются числами, только одна их составляющая. Например 10px. Оператор + для таких значений возвратит NaN. Функция Number.parseInt и ее аналог Number.parseFloat преобразуют строку символ за символом, пока это возможно. При возникновении ошибки возвращается число, которое получилось. Функция Number.parseInt читает из строки целое число, а Number.parseFloat – дробное.

Проверка на число

Для проверки на число можно использовать метод Number.isNaN(str). Он проверяет, является ли указанное значение NaN или нет. Этот метод отличается от глобального isNaN() и отвечает на вопрос "Это Not A Number?".

Всё, что не NaN, при передаче в Number.isNaN вернёт false. Отличные примеры посмотрите тут.

const num = "14.57";
console.log( +num ); // 14.57

console.log( +"12test" ); // NaN

console.log( '10' / "-2" ); // -5

console.log( Number.parseInt('10px') ); //10

console.log( Number.parseFloat('10.5.5') ); // 10.5

console.log( Number.parseInt('a12345') ); // NaN

console.log( Number.isNaN('123px') ); //false

console.log( Number.isNaN('hello' * 'user') ); //true

Сложение чисел с плавающей запятой

Есть очень важная "особенность" при сложении не целых чисел в JavaScript. В кратце 0.1 + 0.2 это не равно 0.3. Результат сложения 0.1 и 0.2 немного больше чем 0.3. В то время как мы считаем в десятичной системе, машина считает в двоичной системе.

Число 0.1 в двоичной системе счисления это бесконечная дробь, так как единица на десять в двоичной системе так просто не делится. Также бесконечной дробью является 0.2. Двоичное значение бесконечных дробей хранится только до определенного знака, поэтому возникает неточность. Когда мы складываем 0.1 и 0.2, то две неточности складываются, получаем незначительную, но всё же ошибку в вычислениях.

Конечно, это не означает, что точные вычисления для таких чисел невозможны. Они возможны. И даже необходимы. Есть несколько методов решения этой проблемы:

  • Сделать их целыми, умножив на 100, сложить, а потом результат разделить на 100
  • Сложить, а затем округлить до разумного знака после запятой. Округления до 10-го знака обычно бывает достаточно, чтобы отсечь ошибку вычислений
console.log( 0.1 + 0.2 === 0.3);

console.log(0.1 + 0.2);

// Сделать их целыми, умножив на 100,
// сложить, а потом результат разделить на 100
console.log( (0.1 * 100 + 0.2 * 100) / 100 );

//Сложить, а затем округлить до разумного знака после запятой
const result = 0.1 + 0.2;
console.log( +result.toFixed(10) );

Строки

Строка (или строчный литерал) - это набор из нуля или больше символов, заключенных в одинарные или двойные кавычки.

const name = 'Mango';

Содержимое строки в JavaScript нельзя изменять. Нельзя взять символ посередине и заменить его. Как только строка создана – она такая навсегда. Можно лишь создать целиком новую строку и присвоить в переменную вместо старой.

Конкатенация строк

Если применить + к строке и любому другому типу данных, результатом операции "сложения" будет строка. Данная операция называется конкатенация, или сложение строк. Во время конкатенации, любой тип данных приводится к строке и сшивается со строкой в выражении сложения, но есть особенность - последовательность записи операндов.

Методы строк

У каждой строки есть встроеные в JavaScript методы. Что такое метод и почему он есть у строк будет более детально разобрано в модуле о объектах. Сейчас мы просто научимся ими пользоваться.

Для использования метода, необходимо поставить точку после строчного литерала или переменной которая содержит строку, после чего написать имя метода.

Индексация символов строки начинается с нуля, не с единицы! Всегда важно об этом помнить. К примеру в строке "JavaScript is amazing!" буква "J" стоит на позиции с индексом 0.

Экранирование

Если строка в одинарных кавычках, то внутренние одинарные кавычки внутри должны быть экранированы, то есть снабжены обратным слешем \'. В двойных кавычках – экранируются внутренние двойные.

Экранирование служит исключительно для правильного восприятия строки JavaScript. В памяти строка будет содержать сам символ без обратного слеша. Экранировать можно любой символ, если он не специальный ничего не произойдёт.

Сам символ обратного слэша \ является служебным, поэтому всегда экранируется, т.е пишется как \\.

console.log("Let's learn JavaScript!"); //Let's learn JavaScript!
console.log('Let\'s learn JavaScript!'); //Let's learn JavaScript!

Шаблонные строки и интерполяция

Шаблонные строки это строчные литералы, допускающие использование выражений. Вы можете использовать многострочные литералы и возможности интерполяции.

Шаблонные строки заключены в обратные кавычки `` вместо двойных или одинарных. Они могут содержать местозаполнители, которые обозначаются знаком доллара и фигурными скобками ${переменная или выражение}.
